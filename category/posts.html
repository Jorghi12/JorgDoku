A look into Python’s Data Model, what it means to be object oriented.

  • Normal Methods
  • Class Methods
  • Static Methods

Defining a method usually look like this

class Pokemon:
    """
    A contrived Pokemon DataModel
    """
    # Class Attributes owned by the Class
    types = ('fire', 'grass', 'rock', 'electric', 'water')

    def __init__(self, name, type, level):
        "Instantiate a pokemon with a name, type and level."
        self.name  = name
        if type not in self.types:
            raise AttributeError('Pokemon not a valid type')
        self.type  = type
        self.level = level

    # Normal Method
    def level_up(self, levels=1):
        self.level += levels
        return self.level
